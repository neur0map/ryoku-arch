After install, the machine reboots into SDDM. Pick the **Ryoku** session (Hyprland) and log in.

## What you see

- **Top: the taskbar.** It shows the Ryoku entry point, active window, workspaces, tray, status icons, clock, and power/session controls when those modules are enabled.
- **Center and drawers: the launcher, island, dashboard, notifications, and utilities surfaces.** They open on demand instead of filling the desktop by default.
- **Background: the wallpaper and desktop widgets.** A default Ryoku wallpaper ships; change it from Settings.
- **No windows yet.** Hyprland is a tiling compositor; nothing is open until you open it. `Mod+T` for a terminal.

`Mod` is `Super` (the Windows / Cmd key) on a normal install.

## Shell surfaces

- **Launcher:** app search, calculator, wallpaper selection, and theme actions.
- **Dashboard:** media, weather, calendar/date, user, and performance panels.
- **Notifications:** grouped notification history and actions.
- **Utilities:** quick toggles, idle inhibit, screen recording controls, and recordings list.

Popup-style shell surfaces can be closed by clicking outside them or by
re-pressing their trigger.

## Common next steps

- **Change the wallpaper.** Use Settings to open the picker. Drops in `~/Pictures/Wallpapers`. Theme colors regenerate from the selected wallpaper unless you've set a static accent.
- **Tune the taskbar.** Settings → Taskbar controls persistence, reveal behavior, workspaces, tray, clock, and status icons.
- **Pick a look.** Settings → Appearance controls mode, color scheme, Material variant, transparency, geometry, font scale, and animation timing.
- **Wire up your VPN.** Use the Ryoku VPN command family and service-backed shell surfaces. See [Plugins](/docs/plugins).
- **Update the shell.** `ryoku-update` in a terminal, or click the update indicator when it appears in the bar.

<Callout type="info">
This build includes an optional security-aware plugin lane, but it is one of many Ryoku modules.
</Callout>

## If the bar isn't there

If sidebars, launcher, or settings fail to draw, drop to a TTY (`Ctrl+Alt+F2`), log in, and run:

```bash
journalctl --user -u ryoku-shell -n 80
hyprctl binds
```

The [troubleshooting guide](/docs/troubleshoot) covers common boot/login issues.
