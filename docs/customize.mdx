## Settings (`Mod+,`)

Most knobs are here:

- **Appearance:** mode, color scheme, Material color variant, transparency, geometry, font scale, and animation timing.
- **Background:** wallpaper source, color generation, visualizer, and desktop widget controls.
- **Taskbar:** persistent/reveal behavior, workspaces, active-window display, tray, clock, status icons, and scroll actions.
- **Island:** media island behavior and audio visualizer controls.
- **Sidebar:** enablement, reveal threshold, panel rounding, and shadow.
- **Lock / Login:** SDDM theme, lock screen layout.
- **System / Services:** update channel, smart color scheme, audio, and service-backed toggles.
- **Hyprland:** compositor-facing settings that Ryoku exposes through the control center.
- **Widgets:** desktop clock, media, resources, weather, battery, and edit-mode controls.

Settings writes through typed Ryoku shell config and command adapters. If a system behavior needs package, service, network, display, power, wallpaper, or hardware changes, it should go through a named `ryoku-*` command rather than a one-off QML side effect.

## Wallpapers

- Use Settings to open wallpaper and background controls.
- Drop files into `~/Pictures/Wallpapers`. The picker auto-discovers new ones.
- The launcher and background controls expose the current wallpaper picker.
- Theme colors regenerate from the selected wallpaper unless you've set a static accent in Settings → Appearance.

Per-monitor wallpapers are configurable in Settings → Background → Multi-monitor.

### Live wallpapers

Ryoku supports three wallpaper types:

| Type | Formats | Backend required |
|------|---------|-----------------|
| Static image | png, jpg, webp, jxl, … | swww (always present) |
| Animated | GIF, animated WebP | `awww` |
| Video | mp4, mkv, mov, webm | `mpvpaper` |

**Setting one:** drop the file in `~/Pictures/Wallpapers` and pick it with `Super+W` (wallpaper picker), or set it from the Launcher. The selection routes through `ryoku wallpaper -f` which detects the file type and hands it to the right backend.

**Settings → Wallpaper → General** exposes a "Live wallpaper" section with:
- **Enable** — turn live wallpaper on or off without losing the path.
- **Mute** — silence audio in video wallpapers.
- **FPS cap** — UI knob, not yet wired to the backend; reserved for a future release.
- **Transition** — swww transition style applied when switching wallpapers.
- **Pause on fullscreen** — pause the live wallpaper when an app goes fullscreen.

**Game mode** (`Super+G`) automatically pauses live wallpapers via the same pause-on-fullscreen toggle.

**NVIDIA / hybrid-GPU note:** `mpvpaper` (video) can misbehave on some NVIDIA and hybrid setups due to DMA-BUF / EGL quirks. `awww` (animated GIF/WebP) is the safer choice on those machines.

## Appearance

Settings → Appearance controls the active mode, generated color scheme,
Material variant, transparency, rounding, padding, spacing, font scale, and
animation timing. Those values flow through the typed config layer and shared
tokens so the taskbar, panels, dashboard, launcher, lock-adjacent surfaces, and
control center stay visually aligned.

## Keybinds

The shipped binds live in `~/.config/hypr/hyprland.lua`. To customize:

- Add your own binds and overrides in `~/.config/hypr/custom.lua`; that file is yours and survives updates. The config is now native Lua (`hl.bind(...)`), not hyprlang.
- Keep a backup before replacing shipped binds; future Ryoku defaults may arrive through migrations or refresh commands.
- Use `docs/keybindings.md` for the documented shipped binding baseline.

## Advanced

For anything that doesn't have a Settings switch:

- **Hyprland layout:** `~/.config/hypr/hyprland.lua` (native Lua); put personal overrides in `~/.config/hypr/custom.lua`. Hyprland docs: https://wiki.hypr.land/
- **Shell config (raw JSON):** `~/.config/ryoku/shell.json`. Settings writes here; you can hand-edit when you know what you want.
- **Shell token overrides:** `~/.config/ryoku/shell-tokens.json`.
- **Theme palette overrides:** `~/.config/matugen/templates/` and `~/.local/state/quickshell/user/generated/`. See [`docs/branding.md`](/docs/branding) for the brand-color anchors.

The [customization inventory](/docs/customization-inventory) is the authoritative list of every text-based surface you can safely touch.
